# LaTeX Generation Instructions for Academic Report

## Purpose

Convert the provided report text into a clean, consistent LaTeX document. Follow these instructions strictly. Do not introduce new styles or formatting decisions.

---

## 0. Continuous Weekly Workflow (MANDATORY)

This report is written over multiple weeks and may be generated by different AI agents.

Use one of these modes each time:

### Initialize Mode

Use this when no LaTeX file exists yet, or when explicitly asked to regenerate everything.

### Update Mode

Use this when a LaTeX file already exists and new text is provided for this week.

Rules for Update Mode:

* Read the existing `.tex` file first.
* Preserve the existing preamble, front matter, and formatting rules.
* Update only the chapters/sections covered by the new input text.
* Keep all other existing chapters unchanged unless explicitly asked to modify them.
* Do not assume fixed chapter names in advance.
* Keep existing chapter titles and order unless the input explicitly changes them.

---

## 1. Document Setup

* Use the following document class:

```latex
\documentclass[12pt,a4paper]{report}
```

* Allowed packages ONLY:

```latex
\usepackage{graphicx}
\usepackage{float}
\usepackage{times}
```

* Notes:

  * The `times` package ensures a Times New Roman-like font.

* Do NOT:

  * Add any other packages.
  * Change margins.
  * Modify document class.

---

## 2. Document Structure

* Use only these sectioning commands:

  * `\chapter{}` for main chapters.
  * `\section{}` for subtopics.
  * `\subsection{}` when needed.

* Rules:

  * Do NOT skip hierarchy levels.
  * Do NOT invent new section types.
  * Preserve logical structure from input text.

---

## 3. Front Matter (MANDATORY)

The document must begin with:

```latex
\begin{document}

\\tableofcontents
\listoffigures

\chapter{...}
```

* `\tableofcontents` generates index automatically.
* `\listoffigures` lists all figures.

---

## 4. Text Formatting

* Write in normal paragraphs.
* Do not use custom formatting (no colors, fonts, spacing tweaks).
* Keep formatting minimal and clean.

### Lists

Use only:

```latex
\begin{itemize}
\item ...
\end{itemize}
```

---

## 5. Figures (Graphics)

### When to include

* Add figures only when they improve understanding.
* Add a figure only when a real image file is provided or clearly specified.
* Do NOT create placeholder image filenames.
* Do NOT overuse figures.

### Mandatory figure format

```latex
\begin{figure}[H]
\centering
\includegraphics[width=0.7\textwidth]{filename}
\caption{Clear and descriptive caption}
\label{fig:short_name}
\end{figure}
```

### Rules

* Always include:

  * `\centering`
  * `\caption{}`
  * `\label{}`
* Width must be between `0.6\textwidth` and `0.8\textwidth`.
* Use simple filenames (example: `diagram1.png`).
* Always reference figures in text.

### Do NOT

* Use subfigures.
* Wrap text around images.
* Use advanced graphics packages (example: TikZ).
* Apply custom styling.

---

## 6. Tables

Use simple tables only:

```latex
\begin{table}[H]
\centering
\begin{tabular}{|c|c|}
\hline
A & B \\
\hline
\end{tabular}
\caption{Caption here}
\end{table}
```

---

## 7. Equations

If needed, use:

```latex
\begin{equation}
...
\end{equation}
```

---

## 8. Output Requirements

* Return a complete LaTeX document.
* Must include:

  * `\begin{document}`
  * `\end{document}`
* Ensure:

  * Table of contents is included.
  * List of figures is included.
* Place content correctly into chapters and sections.
* In Update Mode, return the full updated `.tex` file content, not only partial snippets.
* Maintain consistency with previously generated content.

---

## 9. Strict Constraints

The AI must NOT:

* Add new packages.
* Modify formatting rules.
* Change figure/table styles.
* Introduce custom commands.
* Rewrite unchanged chapters when running in Update Mode.
* Deviate from this instruction file.

---

## 10. Consistency Requirement

* Formatting must remain identical across all chapters.
* Preserve existing labels and references when possible.
* This file is the single source of truth.

---

## 11. Quality Checklist (MANDATORY)

Before final output, verify:

* The document is complete and compile-ready.
* Only allowed packages are used.
* Section hierarchy is valid (chapter, section, subsection).
* No placeholder figures are included.
* No custom commands or style deviations are introduced.
* In Update Mode, unchanged chapters remain unchanged.

---

## Final Instruction

Focus on:

* Clarity
* Consistency
* Simplicity
